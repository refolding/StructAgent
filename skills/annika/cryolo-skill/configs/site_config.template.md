# crYOLO site config — TEMPLATE

This is the **shape** of the local config report. The real report is generated by the
read-only probe and written to `site_config.local.md` (local/private; git-ignored). Do not
fill this template by hand for gated decisions — generate it:

```bash
python3 scripts/cryolo_env_probe.py --format markdown --output configs/site_config.local.md
# machine-readable:
python3 scripts/cryolo_env_probe.py --format json
```

Canonical schema and the `support_assessment` decision rules:
`references/02_config_session_and_environment.md`.

### Safety attestation

This block attests to what the **read-only probe** did while generating the report — the
probe itself runs no crYOLO jobs, so `cryolo_jobs_run` is always `none` here. The **skill**
may run real crYOLO jobs (config/train/predict/evaluation) on a `supported`/`partial`
machine, but only **after explicit user confirmation**; see `SKILL.md §0` and
`references/00_scope_and_trust.md`.

- installs_performed / downloads_performed / network_calls: `none` (probe is read-only)
- cryolo_jobs_run: `none` (the probe never runs jobs; skill execution is gated separately)
- cryolo_exec_enabled: `yes | no` (whether the host's verdict permits the skill to offer
  execution after confirmation — distinct from anything the probe ran)
- commands_run: `[ each external command the probe attempted + status ]`
